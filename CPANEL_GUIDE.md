
# cPanel & Passenger Deployment Guide

Since you are using **cPanel's "Setup Node.js App"** (Passenger), the setup is different from a standard VPS.

## 1. The .htaccess File
I have updated `public/.htaccess`. When you deploy:
1.  **Keep the code `DO NOT RESIZE/CHANGE` block** generated by cPanel at the top of your `.htaccess` (if it exists).
2.  Add my new code **below** it.
3.  **Crucial**: My code removes the `localhost:5000` proxy. Passenger does not need it.

## 2. Environment Variables (The cause of "HTML" Errors/Crashes)
Passenger **DOES NOT** read your locally uploaded `.env` file by default.
You **MUST** set these variables in the cPanel Dashboard.

1.  Go to **cPanel** -> **Setup Node.js App**.
2.  Edit your application.
3.  Scroll to **"Environment Variables"**.
4.  Add the following keys and values (copy from your local `.env`):
    *   `MANI_EMAIL`
    *   `MANI_EMAIL_PASS`
    *   `SUPPORT_EMAIL`
    *   `SUPPORT_EMAIL_PASS`
    *   `SMTP_HOST`
    *   `SMTP_PORT`
    *   `VITE_API_BASE_URL` (Set this to `/api`)

**If you do not do this, `server.js` will crash when sending emails.** When it crashes, Passenger shows an HTML Error Page. React receives that HTML instead of JSON and throws "Unexpected token <".

## 3. Server.js Path Verification
In cPanel, make sure your Node App settings are:
*   **Application Root**: `infolexus` (or wherever you uploaded the files)
*   **Application Startup File**: `server.js`
*   **Run npm install**: Click the "Run NPM Install" button in cPanel to ensure dependencies are compliant with the server's OS.

## 4. Restart
After adding the Environment Variables, click **Restart** in the cPanel Node.js App dashboard.
